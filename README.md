# StrainNode2D

**StrainNode2D** — это 2D-симулятор физики мягких тел и механизмов, построенный на архитектуре узлов и балок (Node-Beam architecture). Движок позволяет создавать сложные конструкции, мосты, автомобили и механизмы, а затем наблюдать за их реалистичной деформацией и разрушением в реальном времени.

Проект вдохновлен такими симуляторами, как Rigs of Rods и BeamNG.drive, перенося концепцию расчетов масс-пружинных сетей в двумерное пространство.

## 🚀 Особенности и технологии (Версия 0.3.0)

- **Реалистичная физика:** Симуляция масс, инерции, гравитации и аэродинамики (включая эффект Магнуса), сухое трение Кулона.
- **Типы балок:** пружина, трос, гидравлика, жёсткая балка, аэробалка — с текучестью и разрывом.
- **Типы узлов:** обычный узел, моторное колесо (`MotorWheel`), структурный узел (`StructuralNode`).
- **Коллизия узлов с балками:** у балки можно включить коллизию и задать радиус — кузов ведёт себя как оболочка, колёса могут ехать по поверхности другого транспорта.
- **Столкновения узлов:** spatial hashing для пар узел–узел и оптимизированный broad-phase для узел–балка.
- **Продвинутое трение:** Честная модель сухого трения Кулона (разделение на статическое трение покоя и кинетическое трение скольжения).
- **Оптимизация столкновений:** Использование Spatial Hashing (пространственного хеширования) для Broad-phase поиска коллизий, что позволяет симулировать сотни узлов без падения FPS.
- **Продвинутый редактор:** Структурное копирование механизмов (с сохранением геометрии), рамка выделения, динамическая камера.
- **Технологический стек:**
  - Язык: `Python 3`
  - Рендер и окно: `Pygame`
  - Диалоговые окна: `Tkinter`

## 📂 Структура проекта

Проект использует стандартную пакетную структуру (Standard Python Layout):
```text
strain-node-2d/
├── strainnode2d/       # Главный пакет приложения
│   ├── core/           # Логика приложения, ввод и камера
│   ├── physics/        # Ядро физического движка
│   └── ui/             # Пользовательский интерфейс (HUD, окна)
├── data/               # Сохранения сцен (.json) и префабы
└── run.py              # Исполняемый файл для запуска
```
## 🎮 Управление

В симуляторе реализован функциональный встроенный редактор для сборки на лету.

### Мышь

| Клавиша / действие | Действие |
|---|---|
| **Колёсико** | Приближение / отдаление камеры |
| **ЛКМ по узлу** | Выделить, перетащить (с инерцией при отпускании) |
| **ЛКМ по балке** | Выделить балку |
| **ЛКМ по пустому месту** | Снять выделение; перетаскивание — перемещение камеры |
| **ЛКМ + Shift по узлу** | Добавить / убрать узел из выделения |
| **ЛКМ + Shift по пустому месту** | Создать случайный узел |
| **ЛКМ + Ctrl** (клик по второму узлу) | Соединить два узла балкой |
| **ПКМ (зажать и тянуть)** | Выделение рамкой |
| **СКМ** | Заморозить / разморозить выделенные узлы |

### Камера и время

| Клавиша | Действие |
|---|---|
| **F** | Следить за выделенным узлом или балкой |
| **Пробел** | Пауза симуляции |
| **↑ / ↓** | Ускорение / замедление времени (0.1× … 1.0×) |
| **Tab** | Режим отладки аэродинамики (выкл → нормали → сопротивление) |

### Редактор и инспектор

| Клавиша | Действие |
|---|---|
| **Q** | Показать / скрыть инспектор |
| **E** | Редактировать свойства выделенного объекта *(инспектор должен быть включён)* |
| **Delete** | Удалить выделенное |
| **Ctrl + C / Ctrl + V** | Структурное копирование узлов и связей между ними |
| **1 – 5** | Сменить тип выделенной балки *(при одной выделенной балке)* |
| **F5 / F9** | Сохранить / загрузить сцену (`.json`) |
| **R** | Перезагрузить последнюю открытую сцену |
| **Insert / Home** | Быстрый чекпоинт / восстановление чекпоинта |

### Механизмы

| Клавиша | Действие |
|---|---|
| **A / D** | Моторы колёс — назад / вперёд |
| **W / S** | Гидравлика — втянуть / выдвинуть |

# 🛠 Запуск проекта

Убедитесь, что у вас установлен Python 3 и библиотека `pygame`.

1. Установите зависимости:
   ```bash
   pip install pygame
   ```
2. Запустите симулятор из корня проекта:
   ```bash
   python run.py
   ```
